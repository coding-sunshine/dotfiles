Produce a concise implementation spec for: **$ARGUMENTS**

Do NOT write code yet. Research the codebase as needed, then output:

1. **Problem** — what we're solving and why (1–2 sentences).
2. **Approach** — the chosen design in a few bullets; note one alternative
   rejected and why.
3. **Changes** — the files/functions to add or modify (reuse existing utilities
   where possible — name them).
4. **Edge cases & risks** — what could break.
5. **Test plan** — how we'll verify it end-to-end.

Keep it scannable. Stop after the spec so we can agree before implementing.
